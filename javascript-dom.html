Introduction to JavaScript fundamentals

Javascript is the crucial piece needed, along with HTML and CSS, that makes possible the dynamic, functional and somewhat intelligent applications that we use in our daily lives. Without Javascript (or a language like it) services like Trademe or Internet Banking couldn’t exist.

For example, Simply adding an item to your watch-list requires that you first have a profile to attribute this watch-list item to. Not to mention display a tick next to it as well as notify the seller that another user has added their item to their watch-list. This is but the tip of the iceberg of complexity here but you get the picture.

Think of it like a game of cards. HTML and CSS could be used to dictate what each card is called, what suit each card is and what they all look like. Javascript however is the language that gives meaning to each card within the game. The underlying logic or rules of the game would be written in Javascript.

One important factor in programming is the order in which things are written. This is known as “Control Flow”. Generally a computer will read code from top to bottom like reading a book. However there are ways in which this can be changed. For instance, you can make the computer repeat a certain section of code a number of times before moving on. This is known as a “Loop”. More on loops in a moment. You can also make the computer choose between one pathway or another depending on a certain condition. This is know as an “if statement”. So I would think of Control flow almost like a toy train set. You can make that train repeat a certain section of track a number of times before it continues back onto the mainline. You could make the train choose one of two different lines to go down depending on how many carriages it is pulling.

Now to Revisit Loops. One handy thing that we can make the computer do is repeat a task a number of times before moving on. This is known as a Loop. In this example I am making the computer count from 1 to 10 and write the number out every time around until it reaches 10. As you can imagine doing this is with a loop is far more convenient than doing it manually. Especially if, for instance, you had to count to 1000.

The DOM could be thought of as “the HTML document from Javascript’s point of view”. Or perhaps “Javascript’s personal map around the HTML file” In order for us to see what javascript is doing it needs to be rendered on the webpage. In order for us to do this we need to manipulate one or more of the HTML elements. For instance change the text inside it, change the colour of it or make it disappear all together. The DOM allows us to do this by giving us a standardised way to tell Javascript which HTML element/s to take hold of and what to do with them.

Here is an example where I tell the javascript to change the color of the h1 text to Blue. The function document.getElementById("heading") means “look for the HTML element with the Identifier of heading” and the “.style.color = "blue”;” means "change the color of the text to blue".

Accessing data from within an array or an object is a common task performed by Javascript. Each data type has their own method for accessing the information within. From within an array you specify what’s known as the ‘index’ of the array or in simpler terms the position of the item in the list. for example: fruits[1] accesses the second item in the list of fruits (array indexes start counting from 0 as apposed to 1).

From within an object it could be thought of as similar to a file path ie “documents/pictures/family-photos/picnic.png” but slightly different syntax. For example: listOfPeople.Steven.eyeColor accesses the eyeColor property from within the Steven object from within the listOfPeople object. you can see how this would be useful when creating applications that are dependant on a set of data.

Every time a customer orders a flat white the last thing you would want to have to do is explain to the Barista each step in detail that is involved in making a flat white. They should already remember those steps and execute them from memory without thinking about it. The same goes for functions in Javascript. Instead of repeating yourself when writing code you can store a set of instructions inside a function and every time that function is “called”, or triggered you could say, the computer will execute those steps. Furthermore this also makes it easier to change the instructions for the function should you need to. As apposed to having to remind them every time to add the sugar. The same goes for functions.
